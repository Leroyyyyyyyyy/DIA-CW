Core Architecture
Market Data Hub
统一接入行情、FX、交易时段、新闻/事件流，输出标准化 MarketSnapshot。所有 agent 都只读这个标准层，不直接各自抓原始源。
Market Trading Agents
做polymarket市场。每个 agent 内部包含 universe scanner、strategy runners、opportunity scorer、position monitor，但它们对外必须只暴露统一接口。
Coordinator / Portfolio Allocator
汇总所有 market agent 的候选机会，按统一评分、风险预算、流动性、冲突约束做横向比较；默认全局同时只允许 1 个活跃机会位，因此它的职责是“在世界范围内只选当前最优的一个”。
Policy + Risk Engine
这是系统的最终裁决层，独立于 LLM 和 Coordinator。它负责市场白名单、品种白名单、最大名义仓位、最大单笔亏损、最大日亏损、最大回撤、滑点阈值、交易时段、冷却期、熔断和 kill switch。
Execution Gateway
将批准后的 TradeIntent 转成 OrderInstruction，交给 Paper Broker Adapter。它负责订单生命周期、幂等键、重试、模拟成交、手续费和滑点模型、持仓回补与对账。
Portfolio State Service
维护系统级持仓、现金、未实现/已实现盈亏、暴露度和风险预算消耗。Coordinator 不直接读 agent 内存状态，只读这里的权威状态。
Event Backbone
运行时统一采用 durable event bus + authoritative database。默认选择：
Postgres 作为交易事实账本与控制面数据存储，
NATS JetStream 作为持久事件总线，
Redis 作为短 TTL 缓存、分布式锁和去重辅助。
衡仓 Dashboard / Ops Console
当前衡仓项目保留并升级为控制台与记录平台，不承担交易主 runtime。它消费交易事实和风险事件，展示资金曲线、持仓、机会列表、订单流、告警、agent 健康、审计轨迹，并承接人工确认、暂停、恢复、关闭工单等管理动作。
